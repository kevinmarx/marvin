# GitHub API patterns

All GitHub operations use `gh` CLI. Never merge PRs — only create drafts, review, and label.

## Create draft PR

```bash
cd {worktree_path}
gh pr create --draft --title "{identifier}: {title}" --body "$(cat <<'EOF'
## Summary

Automated implementation for [{identifier}]({linear_url}): {title}

## Changes

{change_summary}

## Test plan

- [ ] Tests pass locally
- [ ] Code review by human
- [ ] Verify on staging

---
<img src="https://github.com/user-attachments/assets/6dafa5b7-2b93-41da-ad3c-3881d60b7a54" width="20" /> Generated by Marvin
EOF
)"
```

## Capture PR info after creation

```bash
cd {worktree_path}
gh pr view --json url,number -q '.url + " " + (.number | tostring)'
```

## Reply to a review comment

```bash
gh api repos/{repo}/pulls/{pr_number}/comments/{comment_id}/replies \
  -f body="{response_text}"
```

## Resolve a review thread

```bash
gh api graphql -f query='mutation { resolveReviewThread(input: {threadId: "{thread_node_id}"}) { thread { isResolved } } }'
```

Do NOT resolve out-of-scope threads — leave those for the reviewer.

## Fetch PR diff

```bash
gh pr diff {pr_number} --repo {repo}
```

For diff stats only:
```bash
gh pr diff {pr_number} --repo {repo} --stat
```

## Fetch failed CI check runs

```bash
gh api repos/{repo}/commits/{head_sha}/check-runs \
  --jq '.check_runs[] | select(.conclusion=="failure") | {id, name, status, conclusion}'
```

## Fetch check run annotations

```bash
gh api repos/{repo}/check-runs/{check_run_id}/annotations
```

## Fetch Actions job logs

```bash
# Find failed workflow runs
gh api repos/{repo}/actions/runs \
  --jq '.workflow_runs[] | select(.head_sha=="{head_sha}" and .conclusion=="failure") | .id' | head -5

# Get failed jobs
gh api repos/{repo}/actions/runs/{run_id}/jobs \
  --jq '.jobs[] | select(.conclusion=="failure") | {id, name, steps: [.steps[] | select(.conclusion=="failure")]}'

# Fetch log (last 200 lines)
gh api repos/{repo}/actions/jobs/{job_id}/logs 2>/dev/null | tail -200
```

## Create and apply labels (idempotent)

```bash
gh label create "{label_name}" --repo {repo} --color "{hex_color}" --description "{description}" 2>/dev/null || true
gh pr edit {pr_number} --repo {repo} --add-label "{label_name}" 2>/dev/null || true
```

To swap labels:
```bash
gh pr edit {pr_number} --repo {repo} --remove-label "risk:low" --remove-label "risk:medium" --remove-label "risk:high" --add-label "risk:{level}" 2>/dev/null || true
```

## Post a PR review (approve/request-changes/comment)

```bash
gh pr review {pr_number} --repo {repo} --{action} --body "$(cat <<'EOF'
{review_body}
EOF
)"
```

Where `{action}` is one of: `approve`, `request-changes`, `comment`.

## Dismiss a previous review

```bash
REVIEW_ID=$(gh api repos/{repo}/pulls/{pr_number}/reviews \
  --jq "[.[] | select(.user.login==\"{github_user}\" and .state==\"CHANGES_REQUESTED\")] | last | .id")
if [ -n "$REVIEW_ID" ]; then
  gh api repos/{repo}/pulls/{pr_number}/reviews/$REVIEW_ID/dismissals \
    --method PUT -f message="{reason}" -f event="DISMISS"
fi
```

## Safety rules

- **Never merge PRs** — create as draft, undraft only when ready
- **Never force-push** (except `--force-with-lease` after rebase)
- **Never delete branches** — cleanup scripts handle that
